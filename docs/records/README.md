# Records Library
<!-- README.md — Clair's index of the records library. Updated as records are added. -->

This is the root of the records library maintained by Clair, Records Manager for the IT team.
It lives at `E:\Dev\Demo\virtual-team\docs\records\` and is accessible via Claude Code at
`~/.claude/docs/records/`.

---

## Directory Structure

| Folder        | Contents                                                                 |
|---------------|--------------------------------------------------------------------------|
| `technical/`  | Architecture docs, API references, runbooks, system diagrams             |
| `business/`   | Processes, decisions, context, organizational knowledge                  |
| `secrets/`    | Credentials, connection strings, API keys — **role-restricted**          |
| `meetings/`   | Meeting notes, decisions, action items                                   |
| `onboarding/` | Institutional knowledge, team guides, getting-started references         |
| `projects/`   | Project history, status records, deliverable archives                    |

---

## Access Rules

- `technical/`, `business/`, `meetings/`, `onboarding/`, `projects/` — accessible to all team members
- `secrets/` — restricted by role; access granted based on need and cleared by the relevant
  team member responsible for the system in question

---

## Adding a Record

Any team member can add a record by telling Clair what needs to be stored. She will determine
the appropriate location, format it correctly, and confirm placement. Do not add records
directly to this library without going through Clair — consistency of organization is her
responsibility.
